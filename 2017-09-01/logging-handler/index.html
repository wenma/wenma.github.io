python logging模块中将日志重定向到socket流

Sep 1, 2017 | logging | Hits

最近工作中，需要实现一个前端页面实时展示后台任务处理详细过程这么一个功能，类似日志审计，后台任务逻辑和调用关系都比较复杂，审计工作不太容易作为一个独立模块，一开始想到这个功能其实就是往前端页面打日志，想着能不能直接把后台的日志直接输出到前端页面。

要和前端实时交互，不想用ajax被动长轮询的方式的话，那就有两种选择（我至少知道两种）， 一种就是sse, 即server side event, 服务器端事件， 另一种就是websocket. 之前在一个项目里面使用过sse, flask中有一个插件叫flask-sse, 使用挺方便的，代码写起来也挺简单， 但是不好的一点就是需要额外依赖redis, 本身这个系统就是一个小系统，不想额外引入那么多依赖， 所以就想尝试一下用websocket来做。

socket的话自然就想到和stdout, file等都可以抽象成为流，就想着能不能把日志重定向到socket流, 这样就可以通过websocket把日志直接打到web界面上（写这个博客的时候，我可能意识到这个想法就要夭折了，哈哈）。关于重定向，可以先看看最简单的stdout的重定向：
